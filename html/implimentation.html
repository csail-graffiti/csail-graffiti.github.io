Implementation

Graffiti is currently implimented as a centralized system. We chose to make the system centralized to start in order to rapidly prototype with the API it exposes. Eventually we hope that the system is decentralized and end-to-end encrypted. We're pretty certain that both of those things are technically possible but we don't have the resources to impliment it right now. Very open to help.

Graffiti currently has one client side library and it works with the Vue.JS web framework. There is no reason why other frameworks like React, etc. and a nice vanilla JS interface could exist. We just simply don't have the resources. Again, if you want to help out, you can contact us.

Note that there is nothing inherently tying Graffiti to Vue, it just happens to be the first framework we added support for. If you're interested in make a Graffiti plugin for a different web framework, please contact us or make a pull request.
