Media

Media in Graffiti is not stored in the server but instead served peer-to-peer over WebTorrents. You act as the server or seed of your own files and any one who views your files also becomes a seed.

This approach means there is no perscribed limit to the number or size of files you want to share since you are the one responsible for hosting them. The downside is that your media files can go offline when there are no seeders online. We're in the process of creating a seedboxes for Graffiti content, but for now this is simply a limitation of alpha-stage software.

To store media, you can use the asynchronous $gf.media.store function. This function takes as input a File or Blob and returns a magnet link corresponding to your file.

To fetch media, you can use the $gf.media.fetchBlobURL function which takes a magnet link as input and returns a link to a local copy of the object that can be fed into the src of <img> tags, for example.
