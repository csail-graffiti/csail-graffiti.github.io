Vue Plugin Reference

The Graffiti plugin for Vue.JS inherits from the vanilla interface and is mostly identitcal. However some methods have been augmented or added to take advantage of Vue's reactive features.

The plugin exposes the global Vue property $graffiti and it's alias $gf. This property can be accessed via this.$gf in components or as simply $gf Vue templates.

The following properties of $gf are identitcal to their vanilla counterparts:

  • toggleLogIn
  • events
  • post
  • remove
  • objects
  • myContexts

Plugin-Specific Variable

get me → Reactive Actor URI

This is a reactive variation of the vanilla gf.me. The variable is equal to the user's actor URI when they are logged in and connected to the Graffiti server and null otherwise. However, unlike the vanilla version, changes between these states will trigger downstream effects. You can use this to change the description of a toggleLogIn button, for example.

get connected → Reactive Boolean

This reactive variable simplifies access to the connection status that is available only available in the vanilla API via event listeners. When connected to the Graffiti server the variable is true, and false otherwise. For example, try turning your WiFi on and off and observing the demo below:

function useObjects(context)
→ { objects: Reactive Array }

This is a Vue composable wrapper around the vanilla async generator objects function that makes it possible to access Graffiti objects via a reactive array. Any references to the reactive array, either in the template or computed properties, will automatically update when new objects are added by any user. Cleanup is automatically handled when the parent component is destroyed.

Since the setup function that composables must be used within is called before the component is initialized, Graffiti must be injected in but then can be used as normal.
